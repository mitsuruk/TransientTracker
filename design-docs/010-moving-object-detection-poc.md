# TransientTracker Proof of Concept(PoC)方針メモ

## 目的

同じ視野で撮影した複数の天体画像から, 彗星や小惑星のような移動天体候補を抽出する.

## 結論

- 1枚の画像は2次元, 複数枚の画像列は3次元配列 `(frame, y, x)` として扱うのが自然.
- 成否を決めるのは保存形式ではなく, `位置合わせ + 差分検出 + 軌跡追跡` の処理系.
- 位置合わせには必ず誤差が残るため, 完全一致ではなく `誤差で説明できる差分はノイズとして除去する` 方針を採る.
- 最終的には, 残差のうち `時系列で一貫して移動する観測点` を移動天体候補とみなす.

## 基本アイデア

1. 恒星を基準に各フレームを位置合わせする.
2. 各画素の時間中央値から基準画像 `R` を作る.
3. 許容位置ずれ `e` ピクセルを設定する.
4. `R` に対して, 半径 `e` の局所最小画像 `R_min` と局所最大画像 `R_max` を作る.
5. 各フレーム `I_t` について, `R_min <= I_t <= R_max` の範囲に収まる差分は, 位置ずれ由来の静止背景として除去する.
6. 残った差分領域から候補点を抽出する.
7. 候補点をフレーム間で連結し, 速度と方向が概ね一定のものだけを残す.

## 重要な考え方

### 位置合わせ誤差の扱い

- 位置合わせはどれだけ工夫しても完全にはならない.
- そのため, 星像の残差をゼロにするのではなく, 許容幅 `e` の中に入る残差を背景ノイズとして吸収する.
- `e` は固定値で決め打ちせず, 位置合わせ後の基準星の残差分布から決めるのが望ましい.
- `e` を大きくしすぎると, ゆっくり動く天体まで消えるため, できるだけ小さく保つ.

### 移動天体の抽出

- 静止天体は, 位置ずれ許容後の差分処理で大半を落とせる.
- 一方で移動天体は, フレームごとに位置が変わるため, 残差として残る.
- 単発ノイズではなく, 複数フレームで一貫した移動を示すものを候補とする.
- 彗星のように少し広がった像もあり得るため, 点状だけでなく少し広がった候補も許容する.

## PoCで検証すべき項目

- 位置合わせ後の残差から, 実用的な `e` を決められるか.
- 差分後に, 静止星の偽検出を十分に減らせるか.
- 3フレーム以上で連続する移動候補を安定して抽出できるか.
- 暗い対象に対して, 必要なら `shift-and-stack` を追加すべきか.

## PoCのスコープ

- 入力は, 同一視野の時系列画像群.
- 出力は, 移動天体候補の座標列, 輝度指標, 速度推定値.
- 目視確認用として, 先頭星図上に主軌跡を重ねた要約画像も出力対象とする.
- この段階では, 厳密な天体同定や軌道決定までは扱わない.
