# TransientTracker 移動天体解析器 開発方針

## 略語

- CLI: Command Line Interface
- CSV: Comma-Separated Values
- JSON: JavaScript Object Notation
- PoC: Proof of Concept

## 目的

生成済みデータセットを入力として, 位置合わせ誤差を吸収しながら移動天体候補を抽出し, 真値と比較評価できる独立プログラムを実装する.

## 基本方針

- 解析器は `generate` とは別バイナリとして実装する.
- 生成器との接点は `dataset_root` 配下の `frames/`, `labels.csv`, `metadata.json` に限定する.
- 初期PoCは, `2次元並進の位置合わせ`, `中央値基準画像`, `許容誤差付き包絡`, `残差抽出`, `軌跡連結` に絞る.
- 実装は `C++20 + CMake + xtensor + OpenCV` を前提とする.
- 正解率の最適化より, 中間結果が追跡できることを優先する.

## 解析対象

- 入力は生成器が出力した1データセット単位のディレクトリとする.
- 1データセットは, 同一視野, 同一条件の時系列フレーム群を表す.
- 初期実装では, 1データセットあたり移動天体は1個を主対象とする.
- 実画像対応, 複数天体本格対応, 軌道決定は初期スコープ外とする.

## 開発原則

- まず `読める`, `途中結果が見える`, `失敗理由が分かる` 状態を作る.
- 解析処理は, `入力`, `前処理`, `検出`, `追跡`, `評価`, `出力` の層に分ける.
- 各層は, 前段の成果物だけに依存する.
- 生成器内部のレンダリング実装には依存しない.
- 真値比較は評価層でのみ扱い, 検出ロジック本体には混ぜない.

## 初期PoCの到達点

- `dataset_root` を指定して解析を開始できる.
- 各フレームの推定シフトと位置合わせ結果を得られる.
- 基準画像, 包絡画像, 残差画像, 候補マスクを保存できる.
- 候補点をフレーム間で連結し, 軌跡候補を出力できる.
- `labels.csv` と照合して, 検出成否をデータセット単位で評価できる.

## 段階的な開発方針

### Phase 1. データセット読込と検証

- `dataset_root` の構造を検証する.
- `metadata.json` と `labels.csv` を読み, フレーム列を順番に取得する.
- この段階では解析は行わず, 入力契約の安定化を優先する.

### Phase 2. 位置合わせなしの基礎検出

- 読み込んだ画像に対して, 位置合わせなしで中央値基準画像と差分処理を試す.
- `S-01` でパイプライン全体が流れることを確認する.
- この段階の目的は, 候補抽出と出力形式の確定である.

### Phase 3. 並進位置合わせ

- 位相相関または明るい星の重心対応で並進ずれを推定する.
- フレームを共通座標系へ再配置し, 基準画像の品質を改善する.
- `S-02` のようなシフト強めのケースを通す.

### Phase 4. 許容誤差付き差分

- 位置合わせ後画像から中央値基準画像を作る.
- 半径 `e` の包絡画像 `R_min`, `R_max` を構築する.
- 静止星由来の残差を抑え, `S-03`, `S-05` のノイズ条件へ耐性を持たせる.

### Phase 5. 軌跡連結と評価

- 候補点をフレーム間で連結し, 軌跡候補へまとめる.
- `labels.csv` と照合し, 真値近傍の候補を検出成功として評価する.
- 再現率, 偽陽性数, 軌跡長などの指標を出す.

## 優先度

- 最優先は, `読込契約`, `位置合わせ`, `残差抽出`, `軌跡連結`, `真値評価`.
- 次点は, デバッグ画像保存とパラメータ外出し.
- 後回しは, `shift-and-stack`, 回転を含む位置合わせ, 複数天体同時追跡.

## 想定出力

- `summary.json`
- `detections.csv`
- `tracks.csv`
- `evaluation.json`
- `results/result_NNNN.png`
- `results/trajectory_result.png`
- `debug/` 配下の中間画像

## 実装状況 (2026-04-12)

- `Phase 1` から `Phase 5` の最小実装は完了している.
- 位置合わせは `位相相関 + warpAffine` で実装済み.
- 差分は `中央値基準画像 + 半径 e の最大包絡` で実装済み.
- 軌跡連結は隣接フレーム間の最近傍連結で実装済み.
- 真値評価は `labels.csv` と距離閾値で照合する形で実装済み.
- 可視化として, 各フレームの主軌跡画像と, 先頭星図上の `trajectory_result.png` を出力済み.
- 次段階の主課題は, 軌跡スコアの高度化, 複数天体対応, 実データ向けのロバスト化である.

## リスクと対策

- 位置合わせが不安定な場合.
  中間画像と推定シフトを必ず保存し, まず `S-01`, `S-02` で切り分ける.
- しきい値調整が難しい場合.
  グローバル固定値にせず, `MAD(Median Absolute Deviation)` ベースのロバスト推定を使う.
- 偽陽性が多い場合.
  面積制約だけでなく, 軌跡長と直線適合誤差で後段除去する.
