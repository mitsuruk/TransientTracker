# TransientTracker サンプルデータ生成アルゴリズムと開発方針

## 略語

- CLI: Command Line Interface
- PoC: Proof of Concept
- PNG: Portable Network Graphics
- PSF: Point Spread Function
- RNG: Random Number Generator

## 目的

移動天体検出PoCの評価とデバッグに使うため, 正解ラベル付きの人工画像スタックを生成する.

## 基本方針

- 実画像に似せるが, 最初から複雑にしすぎない.
- 背景星, ノイズ, フレームずれ, 移動天体を独立要素として生成し, 段階的に有効化する.
- 検出器の正解判定に使えるよう, 各フレームの真値座標と軌跡IDを必ず保存する.
- 同一パラメータと同一乱数シードで再現できるようにする.
- データ生成器は独立した実行プログラムとして実装し, 解析側は生成済みデータを読む別プログラムとする.

## 生成データの構成

- 画像サイズ: `height x width`
- フレーム数: `num_frames`
- 静止星群
- 背景レベル
- 読み出しノイズと光子ノイズ
- フレームごとの並進ずれ
- 移動天体1個以上
- 必要に応じて欠損領域やホットピクセル

出力は次を最低限含む.

- 画像スタック `I(t, y, x)`
- 真のフレームずれ `(dx_t, dy_t)`
- 移動天体の真値座標 `(x_t, y_t)`
- 軌跡ID
- 使用パラメータ

## 生成アルゴリズム

### 1. 座標系と乱数の初期化

1. 画像サイズ, フレーム数, 乱数シードを決める.
2. 全フレーム共通の理想座標系を定義する.
3. `RNG` を初期化する.

### 2. 静止星群の生成

1. 星数 `num_stars` を決める.
2. 各星について, 真の位置 `(x_i, y_i)` を一様分布または実データに近い分布で生成する.
3. 星の明るさ `flux_i` を広いダイナミックレンジで与える.
4. 各星を `PSF(Point Spread Function)` で画像へ描画する.

初期PoCでは, `PSF` は2次元ガウスで十分とする.

`Star_i(x, y) = flux_i * exp(-((x - x_i)^2 + (y - y_i)^2) / (2 * sigma_star^2))`

### 3. 背景とノイズの生成

1. 一様背景 `background_level` を加える.
2. 必要なら緩い勾配背景を加える.
3. 読み出しノイズとしてガウス雑音を加える.
4. 光子ノイズを近似するなら, 画素値に応じたポアソン雑音を加える.

初期PoCでは, 一様背景 + ガウス雑音だけでもよい.

### 4. フレームごとの位置ずれ生成

1. 各フレームに対し, 真の並進ずれ `(dx_t, dy_t)` を生成する.
2. ずれ量は, 小さなガウス分布または一様分布から与える.
3. 静止星群と背景をそのずれ量で平行移動して, 未位置合わせ画像を得る.

これにより, 位置合わせ処理の評価用真値が得られる.

### 5. 移動天体の生成

1. 初期位置 `(x_0, y_0)` を決める.
2. 速度 `(v_x, v_y)` を決める.
3. 各フレームの真値位置を次で定義する.

`x_t = x_0 + t * v_x`

`y_t = y_0 + t * v_y`

4. 各フレームに移動天体の像を加える.
5. 彗星らしさを出すなら, 星像より少し大きい `sigma_object` または非対称形状を使う.

小惑星相当の点状天体と, 彗星相当の少し広がった天体は, 別シナリオとして切り分けるのがよい.

### 6. オクルージョンと端部処理

1. 天体が画面端へ近づいた場合, 一部欠けることを許容する.
2. 一部フレームで信号を弱くして, 検出困難ケースを作る.
3. 必要なら1フレームだけ欠損させ, 軌跡連結の頑健性を評価する.

### 7. 正解ラベルの保存

各フレームごとに次を保存する.

- フレーム番号
- 真の並進ずれ
- 移動天体の真値中心座標
- 真値強度
- 軌跡ID
- 画面内外フラグ

## 推奨シナリオ

### S-01 最小正常系

- 静止星群あり
- 移動天体1個
- ノイズ弱め
- フレームずれ小

最初にパイプライン疎通を確認するためのケース.

### S-02 位置合わせ誤差評価

- 静止星群あり
- 移動天体1個
- フレームずれ中程度
- ノイズ弱め

`e` の妥当性と背景除去の効き方を見るケース.

### S-03 暗い対象

- 移動天体の輝度を低くする
- ノイズを強める
- 単フレーム差分では厳しい条件にする

`shift-and-stack` の追加要否を判断しやすい.

### S-04 彗星風

- 天体像を少し広げる
- フレームごとに強度を少し変動させる

点状前提の候補抽出が壊れないかを見るケース.

### S-05 失敗系

- 位置ずれを大きくする
- 星を密にする
- ホットピクセルを混ぜる

誤検出と見逃しの境界を確認するケース.

## 開発方針

### 方針1. 段階的に複雑化する

- 最初は `静止星群 + 点状移動天体 + ガウス雑音 + 並進ずれ` のみで始める.
- その後, 背景勾配, 光子ノイズ, 広がった彗星像, 欠損フレームを追加する.
- 一度に複数要因を増やさない.

### 方針2. 真値を最初から保存する

- 検出結果との比較に必要な値を, 画像生成と同時に保存する.
- 後から推定で真値を復元しようとしない.
- 真値と検出結果の対応付けを自動化しやすい形式にする.

### 方針3. シナリオを固定名で管理する

- シナリオごとに意味のある名前を付ける.
- `S-01`, `S-02` のように番号付きで保存する.
- 回帰確認時に, 同じシナリオを繰り返し実行できるようにする.

### 方針4. 乱数シードを外から制御する

- `seed` を設定ファイルまたは `CLI(Command Line Interface)` 引数から指定できるようにする.
- 再現不能な失敗を避けるため, 実行ログに使用シードを必ず残す.

### 方針5. 生成器と検出器を分離する

- サンプルデータ生成コードは, 検出本体から独立したプログラムにする.
- 生成器は, 画像, ラベル, メタデータをデータセットとして書き出す責務だけを持つ.
- 解析器は, 生成済みデータセットを読んで解析する責務だけを持つ.
- 両者の接点は, `frames/`, `labels.csv`, `metadata.json` から成るファイルI/O(Input/Output)契約のみに限定する.
- 検出器の変更で生成器まで壊れないように責務を分ける.

### 方針6. 可視化を標準機能にする

- 各シナリオで, 代表フレームのPNG(Portable Network Graphics)または同等の確認画像を出力する.
- 真値位置を重畳した画像も保存する.
- 数値ログだけでなく, 目視で破綻を検出できるようにする.

## 実装順序

1. 単一フレームの静止星群生成を実装する.
2. 複数フレーム化し, 真の並進ずれを注入する.
3. 点状移動天体を追加する.
4. 正解ラベル保存を追加する.
5. ノイズ, 背景勾配, 欠損ケースを追加する.
6. 彗星風シナリオを追加する.

## 最低限の検証項目

- 生成画像で星と移動天体が目視で確認できること.
- 真値座標が画素範囲と整合していること.
- 同一シードで同一画像が再生成されること.
- 位置ずれ真値と検出された位置合わせ結果を比較できること.
- 移動天体真値と検出軌跡を距離誤差で比較できること.
