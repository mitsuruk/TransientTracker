# TransientTracker Proof of Concept(PoC)アルゴリズム仕様

## 略語

- MAD: Median Absolute Deviation
- PoC: Proof of Concept

## データモデル

- 入力画像スタックを `I(t, y, x)` とする.
- `t` はフレーム番号, `y` は行, `x` は列を表す.
- 位置合わせ後の画像を `A(t, y, x)` とする.
- 基準画像を `R(y, x)` とする.
- 許容位置ずれを `e` ピクセルとする.

## アルゴリズム全体

1. 入力画像を浮動小数点配列へ変換する.
2. 基準フレームを選び, 各フレームの並進ずれを推定する.
3. 各フレームを共通座標系へ再配置して, 位置合わせ済み画像 `A` を得る.
4. 時系列中央値で基準画像 `R` を作る.
5. `R` から許容誤差付き包絡画像 `R_min`, `R_max` を作る.
6. 各フレームの残差画像を計算し, しきい値で候補点を抽出する.
7. 候補点をフレーム間で連結し, 軌跡候補を作る.
8. 軌跡候補をスコアリングし, 移動天体候補として出力する.

## 詳細仕様

### 1. 入力正規化

- 全フレームを同一の数値型へ変換する.
- 必要なら背景レベルを引き, 明るさスケールをそろえる.
- 初期PoCでは, フレームごとの中央値と分位点を用いたロバスト正規化で十分とする.

### 2. 位置合わせ

- PoCのずれモデルは2次元並進のみとする.
- 各フレーム `I_t` について, 基準フレーム `I_ref` へのずれ `(dx_t, dy_t)` を推定する.
- 実装候補は位相相関または明るい恒星重心の対応付けとする.
- 推定後, 線形補間で再サンプリングして `A_t` を生成する.

### 3. 基準画像生成

- 基準画像は時間中央値で定義する.

`R(y, x) = median_t A(t, y, x)`

- 移動天体が全フレームの少数にしか現れない前提では, 中央値は静止背景を代表しやすい.

### 4. 許容誤差付き包絡画像

- 位置合わせ誤差を吸収するため, `R` に対して半径 `e` の近傍最小値と近傍最大値を取る.
- 近傍集合を `N_e(y, x)` とすると, 次で定義する.

`R_min(y, x) = min_{(u, v) in N_e(y, x)} R(u, v)`

`R_max(y, x) = max_{(u, v) in N_e(y, x)} R(u, v)`

- 実装上は, 半径 `e` の正方窓または円形窓によるモルフォロジ演算として扱える.
- 直感的には, `e` ピクセル以内の位置ずれで説明できる静止天体の明るさ範囲を表す.

### 5. 残差計算

- 明るい移動天体を検出する主経路として, 正の残差を使う.

`D_t(y, x) = A_t(y, x) - R_max(y, x)`

- `D_t` が正で大きいほど, 静止背景と位置ずれでは説明しにくい.
- ノイズ尺度 `sigma_t` は, 画像全体または局所窓に対するロバスト推定で求める.
- 初期PoCでは, `D_t` の背景画素に対する MAD(Median Absolute Deviation) から `sigma_t` を近似してよい.

### 6. 候補点抽出

- 候補マスクを次で定義する.

`M_t(y, x) = 1 if D_t(y, x) > k * sigma_t else 0`

- `k` はしきい値倍率で, 実験パラメータとする.
- `M_t` に連結成分解析を適用し, 候補領域を求める.
- 各候補について, 重心, 最大値, 総和強度, 面積, 外接矩形を算出する.
- 面積が極端に小さいものや大きいものは除外する.

### 7. 軌跡連結

- フレーム `t` の候補 `c_t` とフレーム `t+1` の候補 `c_{t+1}` を, 距離制約で接続する.
- 接続条件は次を基本とする.
  - 変位が最大速度 `v_max` 以下.
  - 方向変化が閾値以下.
  - 強度変化が極端でない.
- 3フレーム以上つながる候補列を軌跡候補とする.
- 軌跡ごとに, 直線近似の残差, 平均速度, 継続長を計算する.

### 8. 候補スコアリング

- 軌跡長が長いほど高評価とする.
- 直線運動への適合度が高いほど高評価とする.
- フレームごとの強度変動が小さいほど高評価とする.
- 最終候補はスコア順に並べる.

## パラメータ

- `e`: 位置合わせ誤差の許容幅.
- `k`: 残差のしきい値倍率.
- `v_max`: 隣接フレーム間の最大移動量.
- `area_min`, `area_max`: 候補領域の面積制約.
- `track_len_min`: 最小軌跡長. 初期値は3を推奨する.

## 失敗しやすいケース

- 位置合わせ誤差が大きく, 静止恒星残差が `e` を超える場合.
- 移動天体が非常に遅く, 中央値画像に写り込む場合.
- 移動天体が暗く, 単フレーム差分ではしきい値を超えない場合.
- 彗星のように広がった像が, 点状前提の面積制約で落ちる場合.

## 拡張候補

- `e` を画面全体で固定せず, 星像分布に応じて局所的に変える.
- 暗い対象向けに `shift-and-stack` を追加する.
- 並進だけでなく, 回転と拡大縮小を含む位置合わせへ拡張する.
